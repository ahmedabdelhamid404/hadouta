### 3. MCP setup attempt — ultimately deferred

Plan was to wire Sentry + PostHog MCPs so chat-driven analytics queries work without dashboard click-through. Reality: hit a stack of OAuth bugs (Claude Code [#12077](https://github.com/anthropics/claude-code/issues/12077)) and auth-realm mismatches that consumed ~1.5 hours.

What we tried and what failed:
- **OAuth flow** (`claude mcp add` + `/mcp`) for both: rejected by Sentry + PostHog with "Invalid redirect URI"
- **Bearer token (Sentry MCP)**: rejected with `invalid_token` — Sentry's MCP gateway is a separate auth realm from their REST API; user-created auth tokens don't work there
- **Bearer token (PostHog MCP)**: token IS accepted, but the JSON-RPC handshake fails with "Mcp-Session-Id header is required" — likely another Claude Code bug

What works fine: REST API access with the same tokens. Pivoted to a runbook-driven approach.

### 7. Phase 2 — design tokens (brand brief v1.1 → globals.css)

`globals.css` rewritten as a 3-tier token system:
- **Tier 1 — Hadouta raw brand palette**: `--hadouta-cream` / `--hadouta-terracotta` / `--hadouta-ochre` / `--hadouta-teal` / `--hadouta-brown` / `--hadouta-blush` (all in oklch — Phase 3 Figma may refine values; oklch is perceptually-uniform so adjustments compose cleanly).
- **Tier 2 — shadcn semantic mapping**: `--background` / `--primary` / `--secondary` / `--muted` / `--accent` / etc. mapped to the brand palette. All existing shadcn components inherit automatically — terracotta CTAs, cream surfaces, brown text, teal accents.
- **Tier 3 — exposed via `@theme inline`**: Tailwind utilities like `bg-hadouta-terracotta`, `text-hadouta-brown` available at component level for cases where semantic-name doesn't fit.

WCAG AA contrast verified across the palette:
- Brown on cream (body): ~12.6:1 — AAA all sizes
- Cream on terracotta (CTA): ~5.0:1 — AA Large (button text qualifies as Large)
- Cream on teal (accent): ~5.0:1 — AA Large
- Brown on ochre (warm band): ~6.5:1 — AA Normal
- Brown on blush (kid-magic): ~9.5:1 — AAA
- Edge case documented in token comments: terracotta is a CTA + bold-display color, not a body-text background. Body content on warm bands uses ochre or blush.

Radius scale per brand brief (avoid both no-radius corporate and pill-radius childish-bubbly):
- `--radius-tight` 4px (chips, badges)
- `--radius-lg` 8px (buttons, inputs — base)
- `--radius-xl` 16px (cards, larger surfaces)
- `--radius-2xl` 24px (modals, sheets, hero panels)

Motion timing per brand brief (storyteller-paced, slower than typical SaaS):
- `--motion-quick` 200ms (button hover, link, micro-interactions)
- `--motion-paced` 400ms (modal open/close, drawer, section reveal)
- `--motion-page` 600ms (page-turn-feel, big state transitions)

Typography stack added per brand brief 3-tier system. `layout.tsx` now imports four fonts via `next/font/google`:
- **Tajawal** — body & UI (kept; loaded since session 2)
- **El Messiri** — general headers (Egyptian-designed humanist Arabic; carries Egyptian visual heritage in the type itself)
- **Aref Ruqaa** — decorative / kid-magic / Hadouta logotype (calligraphic; brand-brief rule: appears no more than once per page)
- **Fraunces** — Latin companion (modern serif, warm-old-book feel; replaces Inter from earlier scaffold)

Each font exposed as a CSS variable (`--font-tajawal`, `--font-el-messiri`, `--font-aref-ruqaa`, `--font-fraunces`) and as Tailwind utilities (`font-sans`, `font-heading`, `font-display`, `font-latin`).

Build verified: `pnpm typecheck` clean, `pnpm build` succeeds, all 4 fonts loaded in production bundle.

Dark mode kept as a stub (mirrors light-mode values in shadcn variables) — no current product requirement; revisit when an admin dashboard ships.

## Notes / observations

- **The MCP debugging odyssey (Sentry + PostHog) was the biggest time-sink of the session.** ~1.5 hours invested in a path that ultimately got deferred. The signal: when integration debugging exceeds the integration's value-per-effort, ship the workaround and move on. The runbook + scripts deliver 80% of the chat-query value without depending on Claude Code's currently-broken MCP OAuth client. Worth treating as a heuristic going forward.
- **Token-rotation hygiene was exercised end-to-end this session.** Two leaked tokens identified (in chat transcript), revoked at provider, regenerated, replaced in `.env.local`. The user did the rotation out-of-band — fastest path. Pattern is now codified in `docs/operations/observability-runbook.md`.
- **The 3-tier design-token system (raw palette → semantic shadcn names → Tailwind utilities) is more setup work than a 2-tier system, but pays off on every single component.** Rebrand happens at tier 1; semantic-name components stay stable. Phase 3 Figma can refine the tier-1 oklch values if needed without rewriting any components. This is the architectural-from-day-one principle from ADR-019, applied to design.
- **The "EU vs US PostHog default bug" caught in this session would have silently lost every event in production** had we deployed without Vercel env vars set. The default was wrong; the env var override saved us. This is the kind of bug where the *fix* is one line, but *finding* it requires actually checking traffic — which is exactly what observability tools are for. Bootstrap-time observability paid for itself before we shipped any user features.
- **Three commits across both sub-repos plus three on the umbrella in one session** is a higher-than-usual landing rate. Reflects that this session was largely *closing-out* work from previous sessions (ADR-018 backend impl, observability deferred from Sprint-1 acceptance, Phase 2 tokens deferred from brand brief). Future sessions should taper as we move into the more-iterative design + frontend implementation phases.
